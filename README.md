## 📂 저장소(Repository) 구조 및 역할

우리 프로젝트는 독립성과 효율적인 백업을 위해 총 4개의 파트별 저장소로 나누어 운영됩니다. 본인 파트 외의 코드가 필요할 때 각 주소에서 수시로 확인하고 가져다 쓰시면 됩니다.

1. 🌡️ **thermal** (열화상 파트)
   - **주요 내용:** 열화상 카메라 데이터 수집, 이미지 전처리 및 시각화 코드
2. 📡 **radar** (레이더 파트)
   - **주요 내용:** 레이더 신호 처리(Signal Processing), Raw data 파싱 스크립트
3. 🧠 **algo-general** (범용 알고리즘 파트)
   - **주요 내용:** 특정 환경에 종속되지 않는 핵심 탐지/분석 로직 소스 코드
4. 🌊 **algo-testbed** (실제 수조 알고리즘 파트)
   - **주요 내용:** 실제 수조 실험 장비/센서 연동 코드, GUI 제어 프로그램 및 실시간 시뮬레이션

---


### 💬 커밋 메시지 규칙 
저장(Commit)할 때 제목 맨 앞에 아래 태그를 붙여 다른 팀원도 한눈에 알 수 있게 합니다.
- `[ADD]`: 새로운 기능 추가
- `[FIX]`: 버그 수정
- `[WORK]`: 아직 작업 중인 코드 (중간 확인 및 백업용)
- `[DOCS]`: 문서 수정 (README 등)
- '[TEST]': 실험 진행 및 테스트 결과 기록

> **💡 예시:** `[WORK] 열화상 데이터 수집 스크립트 작성 중 (피드백 요청)`

---

## 💡 수조 실험(Testbed) 진행 시 
- 현장 수조 실험 중 수정된 코드는 그날 반드시 **`algo-testbed`** 저장소에 날짜와 실험 결과를 기록하여 업로드. (예: `[TEST] 5/12 수조 1차 테스트 에러 수정본`)
